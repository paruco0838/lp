# エアコンクリーニングLP設計図

**目的**: インスタグラム広告からの流入ユーザーを、エアコンクリーニングの「予約（LINE経由）」へ確実につなげる。
**ターゲット**: 20〜30代の子育て中のママ、健康に気を使っている女性。
**デザイン方針**: モバイルファーストでレスポンシブ対応。清潔感・信頼感を重視した「白と青」ベースの配色。

## 全体共通（UI/UX）
- **配色**: ベースカラー（白）、メインカラー（爽やかな青・水色）、アクセントカラー（CTAボタン等にオレンジや黄色で目立たせる）。
- **追従型CTAボタン**: スマホの画面下部（PCでは右下など）に常に「LINEで簡単見積もり・予約」のボタンを固定表示。
- **レスポンシブ設計**: スマホ（縦幅が長くスクロール中心）で見やすく、PCでも崩れないレイアウト。

## 1. ファーストビュー（FV）
- **キャッチコピー**: 例「エアコンのイヤなニオイ、実はカビかも？プロのクリーニングで、赤ちゃんも安心なキレイな空気に！」
- **ビジュアル**: 綺麗なエアコンと赤ちゃんと笑顔のママ、または清潔感あるリビングの写真。青空のイメージ等。
- **フック（惹きつけ）**: 「インスタ限定！防カビ抗菌コート無料プレゼント🎁」
- **メインCTA**: 目立つLINEの緑色（またはアクセントカラー）での「LINEで今すぐ相談・予約する」ボタン。

## 3. 私たちが選ばれる理由（サービスの強み・特徴）
- **特徴1：安心のエコ洗剤**（小さな子どもやペットがいても安心な、人にも環境にも優しい洗剤を使用）
- **特徴2：プロの徹底分解・高圧洗浄**（素人では届かない奥の汚れやカビを、専用の高圧洗浄機で一掃）
- **特徴3：安心の補償制度**（万が一の事前・事後トラブルに備え、損害保険加入済み）
- **特徴4：明朗会計**（汚れ具合による追加料金は一切なし）

## 6. 料金プランと【インスタ限定オファー】
- **通常料金表**: 壁掛けタイプ、お掃除機能付きタイプなどの分かりやすい価格表。複数台割引などがあれば記載。
- **インスタ広告限定オファー**: 「このページからLINE登録＆ご予約いただいた方限定！**防カビ抗菌コート（通常〇〇円）を無料サービス！**」

## 7. ご予約から当日までの流れ
1. **お問い合わせ**: 「LINEで友だち追加」後、自動メッセージに従って希望日やエアコンの型番等を入力。
2. **日程調整のご連絡**: スタッフよりLINEで日程確定のご連絡。
3. **ご訪問・作業**: 作業時間は約〇〜〇時間。事前の準備（家具の移動など）の有無を記載。
4. **ご確認・お支払い**: 綺麗になったことを一緒に確認し、お支払い（現金/PayPay/クレカ等）。

## 11. 【重要】セキュリティと安全な公開のための設計指針
初心者がそのままインターネット上に公開しても、ハッキングや情報漏洩のリスクがないように、以下のセキュリティルールを徹底して設計・構築します。

- **APIキー等のハードコード（直書き）の完全禁止**
  - LINE Messaging APIやGoogle Maps APIなど、外部サービスと連携するための「秘密の鍵（APIキー）」を、HTMLやJavaScriptのファイル内に直接書き込むことは**絶対にしません**。
  - フロントエンド（ブラウザで見える部分）のコードに書かれたキーは、誰でも簡単に盗み見て悪用できるためです。
  - **対策**: もしAPIを利用する場合は、バックエンド（サーバー側）を用意するか、Netlify FunctionsやCloudflare Workersなどの「サーバーレス関数」を経由させる、もしくは環境変数（`.env`ファイルなど）で管理して本番環境に安全に渡す仕組みを作ります。（今回は静的HTMLとLINE公式アカウントへの直接リンクをメインとするため、原則としてフロントエンドでAPIキーを扱う要件は発生させません）
- **静的サイトによる堅牢性の確保**
  - WordPressのようなCMS（裏側でデータベースと連携し動的にページを作るシステム）を使わず、HTML/CSS/JSのみの**「静的サイト」**として構築します。
  - これにより、データベースへの攻撃（SQLインジェクション等）や、管理画面のパスワードを破られて改ざんされるリスクを根本から排除できます。
- **安全なフォームの実装（必要な場合）**
  - 予約をLINE公式アカウント（友だち追加URLへのリンク）で完結させることで、自前で個人情報（名前・電話番号等）を収集するフォームやサーバーを持つ必要がなくなり、情報漏洩リスクを大幅に下げます。
  - 万が一、自前でWebフォーム（お問い合わせ等）を設置する場合は、初心者でも安全に使える「Netlify Forms」や「Googleフォームの埋め込み」など、外部の信頼できるサービスを利用し、自前でメール送信プログラム（PHPなど）を書くことは避けます。
- **常時SSL化（HTTPS）の必須化**
  - サイトを公開する際は、通信内容を暗号化する「HTTPS（SSL証明書あり）」の環境で公開することを前提とします（例: Netlify, Vercel, GitHub Pagesなどは標準で適用されます）。
